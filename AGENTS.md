# 编码风格约定

这个仓库里的前端代码请优先保持已有风格，不要擅自做整文件格式化。

## Template

- 标签属性尽量写在一行。
- 只有超过编辑器正常可视范围时才换行。
- 换行后不要切得太碎，尽量保持一行至少 3 个属性。

## Import

- `import ... from ...` 的 `from` 需要对齐。
- 不要把已经对齐好的 import 改乱。

## 箭头函数

- 单参数箭头函数使用 `x => xxx`，不要改成 `(x) => xxx`。
- 只有 0 个参数或多个参数时才使用括号。

## 注释与缩进

- 注释使用中文注释。
- 缩进使用 4 个空格，除非文件已有明确且一致的其他风格。

## 修改原则

- 只在完成任务所需的最小范围内调整格式。
- 不要顺手重排整文件。
- 新代码优先贴合周围已有写法。
