# 快速创建插件工程

推荐把插件开发拆成两层：

- `@ddm-center/create-plugin`：创建 mixed 插件工程，默认同时包含主进程入口、渲染进程入口和 i18n 入口。
- `@ddm-center/plugin-api`：只提供宿主 API 类型声明，不打包宿主实现。

## 创建工程

```bash
npm create @ddm-center/plugin my-plugin
```

也可以显式指定 JavaScript：

```bash
npm create @ddm-center/plugin my-plugin -- --js
```

生成的插件名会自动补成 `ddm-plugin-my-plugin`，目录结构如下：

```text
ddm-plugin-my-plugin/
    src/main/index.js
    src/renderer/index.js
    src/renderer/views/Settings.vue
    src/i18n/index.js
    package.json
    webpack.config.js
    jsconfig.json
```

## mixed 插件入口

主进程入口导出函数，通过 `appMain` 获取主进程宿主对象：

```js
import appMain from 'appMain';

export default function installMain() {
    return {
        async install() {
            appMain.exportPluginService('helloService', {
                sayHello(name) {
                    return `Hello ${name || 'DDM'}`;
                },
            });
        },
    };
}
```

渲染进程入口导出函数，通过 `appView` 获取渲染进程宿主对象：

```js
import appView from 'appView';

export default function installRenderer() {
    return {
        async install() {
            const service = appView.referenceService('helloService');
            const message = await service.sayHello('Plugin');
            appView.notify({ title: 'Plugin', body: message });
        },
    };
}
```

## 类型提示

插件工程安装 `@ddm-center/plugin-api` 后，`appMain` 和 `appView` 这类宿主注入模块都由声明文件提供提示。运行时对象仍然由 DDM 宿主注入，插件包不要把宿主实现打进自己的产物。

JavaScript 工程默认使用 `jsconfig.json` 做路径映射：

```json
{
    "compilerOptions": {
        "paths": {
            "appMain": ["node_modules/@ddm-center/plugin-api/appMain.d.ts"],
            "appView": ["node_modules/@ddm-center/plugin-api/appView.d.ts"]
        },
        "types": ["@ddm-center/plugin-api"]
    }
}
```

当前脚手架暂不支持创建 TypeScript 工程。

## 发布工具包

先发布类型包，再发布脚手架：

```bash
npm run publish:plugin-api
npm run publish:create-plugin
```

也可以用一个命令按顺序发布两者：

```bash
npm run publish:plugin-tools
```

两个包都是 scoped package，所以发布命令默认带 `--access public`。
