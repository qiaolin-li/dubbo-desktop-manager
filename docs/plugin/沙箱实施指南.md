# 插件沙箱实施指南

## 快速开始

### 1. 安装依赖（如果需要 VM2）

```bash
npm install vm2 --save
```

### 2. 修改 PluginLoader 集成沙箱

在 `src/main/plugin/PluginLoader.js` 中：

```javascript
import PluginSandbox from '@/main/plugin/sandbox/PluginSandbox.js';

class PluginLoader {
    async load(pluginName, source = null) {
        try {
            const appPlugin = new AppPlugin(pluginName, source);
            // ... 现有代码 ...
            
            // 创建沙箱环境
            const pluginConfig = appConfig.getProperty(`pluginConfig.${pluginName}`) || {};
            const sandbox = new PluginSandbox(appPlugin, pluginConfig);
            const sandboxedPlugin = sandbox.getSandboxedContext();
            
            // 使用沙箱化的插件对象
            if(appPlugin.mainPath){
                const plugin = this.requirePlugin(appPlugin.mainPath)(sandboxedPlugin)
                await plugin.install() 
            }
            
            // 保存沙箱引用
            appPlugin._sandbox = sandbox;
            
            return appPlugin;
        } catch (error) {
            // ... 错误处理 ...
        }
    }
}
```

## 插件权限配置

### 在 package.json 中声明权限

```json
{
  "name": "ddm-plugin-example",
  "version": "1.0.0",
  "ddm": {
    "permissions": {
      "filesystem": {
        "read": ["./"],
        "write": ["./data"]
      },
      "network": {
        "allowedDomains": ["api.example.com", "*.github.com"],
        "maxRequestsPerSecond": 5
      },
      "apis": [
        "datasource.add",
        "datasource.list",
        "config.get"
      ]
    }
  }
}
```

### 在应用配置中设置权限

```javascript
// 允许插件访问特定 API
appConfig.setProperty('pluginConfig.ddm-plugin-example', {
    permissions: {
        apis: ['datasource.*', 'config.get']
    }
});
```

## 兼容性处理

### 向后兼容模式

如果插件没有声明权限，默认行为：
- 文件系统：只能访问插件目录
- 网络：允许所有域名（但有限流）
- API：允许所有 API 调用

### 渐进式迁移

1. **阶段一**：对新插件启用沙箱，旧插件保持兼容
2. **阶段二**：提示用户更新旧插件配置
3. **阶段三**：强制所有插件使用沙箱

## 故障排查

### 插件无法访问文件

1. 检查文件路径是否在允许列表中
2. 查看日志中的权限拒绝信息
3. 确认插件权限配置正确

### 插件网络请求失败

1. 检查域名是否在白名单中
2. 检查请求频率是否超限
3. 查看网络拦截器日志

### API 调用被拒绝

1. 检查 API 路径是否在权限列表中
2. 确认插件已正确声明权限
3. 查看 API 权限检查日志

## 安全建议

1. **最小权限原则**：只授予插件必需的权限
2. **定期审计**：检查插件的权限使用情况
3. **用户确认**：安装插件时让用户确认权限
4. **权限撤销**：支持动态撤销插件权限
5. **日志记录**：记录所有权限检查结果
