# 插件沙箱方案详细设计（单一真相来源）

> 本文档已将原先分散在 `沙箱方案增强.md`、`沙箱方案增强-延迟require问题.md` 的内容全部合并进来。  
> 后续请只维护本文件；其它“增强”文档只保留索引说明，避免重复与不一致。

## 一、目标与威胁模型

### 1.1 目标

- **最小权限**：插件只能通过宿主授予的能力对象做事（capability-based）。
- **可渐进落地**：兼容旧插件，允许逐步收紧默认策略。
- **可观测/可审计**：对被拒绝的访问（文件、网络、require、API）记录日志。

### 1.2 主要威胁

- **绕过能力对象**：插件直接 `require('fs')` / `require('child_process')` 获得系统权限。
- **延迟 require 绕过**：插件在 `setTimeout/Promise.then` 中再 `require('fs')`，如果你用“临时替换 require”，会绕过。
- **网络外联**：插件向任意域名发送请求，泄露敏感数据。
- **滥用主进程 API**：插件调用未授权的模块/方法执行敏感操作。

---

## 二、方案对比（总览）

1. **方案一：API 代理沙箱 + require 持久化拦截（推荐，工程可落地）**
2. **方案二：VM2 沙箱（原理可参考；VM2 已废弃，不建议新上）**
3. **方案三：Worker Threads/独立进程隔离（安全更强，工程成本更高）**

---

## 三、方案一：API 代理沙箱 + require 持久化拦截（推荐）

> 方案一的“最终形态”必须包含两条线：  
> - **A：能力对象限制**（受限 fs/axios/API 权限）  
> - **B：require 限制**（并且必须能拦截延迟 require）

### 3.1 组件与职责（角色清单）

- **`PluginSandbox`**：读取权限配置，组装给插件的能力对象 `sandboxedPlugin`（冻结/只读）。
- **`RestrictedFS`**：路径白名单 + 读写分级（建议：应用数据目录只读，插件目录读写）。
- **`RestrictedAxios`**：域名白名单 + 频率限制（必要时可加代理、证书策略、大小限制）。
- **`RestrictedModuleLoader`**：实现“受限 require”（内置模块白名单、外部包白名单、危险模块黑名单、模块替换）。
- **`ModuleLoadInterceptor`（关键）**：持久化拦截 `Module._load`，把“插件文件触发的 require”全部导向 `restrictedRequire`，**包括延迟 require**。

对应代码文件（已在工程内）：
- `src/main/plugin/sandbox/PluginSandbox.js`
- `src/main/plugin/sandbox/RestrictedAxios.js`
- `src/main/plugin/sandbox/RestrictedModuleLoader.js`
- `src/main/plugin/sandbox/ModuleLoadInterceptor.js`
- （`RestrictedFS.js` 建议补齐落地实现；目前你仓库里该文件可能还没成功落盘）

### 3.2 为什么不能“临时替换 `Module.prototype.require`”

下面这种方式**有致命漏洞**：

```js
Module.prototype.require = restrictedRequire;
require(pluginMain);
Module.prototype.require = originalRequire;
```

插件可通过延迟执行绕过：

```js
setTimeout(() => {
  const fs = require('fs'); // 恢复后：使用原始 require，绕过！
}, 100);
```

因此 require 限制必须是**持久化拦截**：推荐拦截 `Module._load`，并根据 `parent.filename` 判断是否属于插件目录。

### 3.3 权限模型（建议结构）

插件 `package.json` 建议声明：

```json
{
  "ddm": {
    "permissions": {
      "network": {
        "allowedDomains": ["api.example.com", "*.github.com"],
        "maxRequestsPerSecond": 5
      },
      "apis": ["datasource.*", "config.get"],
      "require": {
        "builtins": ["path", "url", "util", "crypto", "events"],
        "packages": ["lodash", "dayjs"]
      }
    }
  }
}
```

策略建议：
- **默认禁止危险内置模块**：`fs`, `child_process`, `net`, `http`, `https`, `worker_threads`…
- **`fs` 走替换**：`require('fs')` 返回 `plugin.fs`（受限实现）。
- **外部包默认禁用**：`require.packages` 为空时，禁止所有外部 npm 包（更安全；需要时逐个放行）。

## 四、方案一：关键实现细节（落地要点）

### 4.1 `RestrictedModuleLoader`（受限 require）

核心能力：
- **builtin 白名单**：只允许极少数安全模块（`path/url/util/...`）。
- **外部包白名单**：默认拒绝，按需放行。
- **危险模块黑名单**：遇到直接抛错（或替换为受限能力）。
- **模块替换**：例如把 `fs` 替换为 `plugin.fs`。

实现文件：
- `src/main/plugin/sandbox/RestrictedModuleLoader.js`

### 4.2 `ModuleLoadInterceptor`（拦截延迟 require 的关键）

实现目标：
- 安装一次全局拦截器：替换 `Module._load`（保留原始引用）。
- 若 `parent.filename` 在某个 `pluginDir` 下：使用该插件注册的 `restrictedRequire`。
- 否则：走原始 `Module._load`。

实现文件：
- `src/main/plugin/sandbox/ModuleLoadInterceptor.js`

### 4.3 PluginLoader 集成（最终推荐流程）

推荐加载流程（伪代码）：

```js
// 1) 应用启动时（只做一次）
ModuleLoadInterceptor.install();

// 2) 加载插件
const sandbox = new PluginSandbox(appPlugin, pluginConfig);
const sandboxedPlugin = sandbox.getSandboxedContext();

const moduleLoader = new RestrictedModuleLoader(
  appPlugin.pluginDir,
  pluginName,
  pluginConfig.permissions?.require || {}
);

// 让 require('fs') 返回受限 fs（而不是原生 fs）
moduleLoader.setModuleReplacement('fs', sandboxedPlugin.fs);

const originalRequire = Module.prototype.require;
const restrictedRequire = moduleLoader.createRestrictedRequire(originalRequire);

// 3) 注册（持久化），保证延迟 require 也会被拦截
ModuleLoadInterceptor.registerPlugin(appPlugin.pluginDir, restrictedRequire);

try {
  delete require.cache[require.resolve(appPlugin.mainPath)];
  const mod = require(appPlugin.mainPath);
  const installable = typeof mod === 'function' ? mod(sandboxedPlugin) : mod;
  await installable.install?.();
} catch (e) {
  // 若加载失败，回滚注册
  ModuleLoadInterceptor.unregisterPlugin(appPlugin.pluginDir);
  throw e;
}

// 4) 卸载插件时注销
ModuleLoadInterceptor.unregisterPlugin(appPlugin.pluginDir);
```

### 4.4 API 权限（主进程 API 调用控制）

现状：`ApiExportor.invokeMethod(moduleName, method, args)` **没有鉴权**。

建议（比“解析调用栈”可靠）：
- 在 IPC/HTTP 请求里**显式携带调用者身份**（例如 `callerPluginName`）。
- `ApiExportor.invokeMethod` 中调用 `PluginSandbox.checkAPIPermission(moduleName, method)` 校验。

> 仅靠解析调用栈找 caller plugin 不可靠；建议走显式字段/通道隔离。

## 五、方案二：VM2 沙箱（不建议新上）

### 5.1 是否能限制 `require('fs')`？

能。VM2 支持：
- `require.builtin` 白名单
- `require.external` 控制外部包
- `require.mock` 替换模块（如替换 `fs`）

### 5.2 为什么有 10-20% 性能开销？

常见来源：
- 代码解析/编译与沙箱上下文创建
- Proxy/陷阱带来的属性访问与调用开销
- 跨上下文数据传递
- 额外安全检查

更关键：**VM2 已废弃且历史上多次出现逃逸漏洞**，建议只当原理参考。

---

## 六、方案三：Worker Threads/独立进程隔离

安全更强（天然隔离），但成本更高：
- 通信需要序列化/RPC 化
- 生命周期管理复杂
- 插件 API 需要重新设计

---

## 七、推荐落地路径

### 7.1 第一阶段（先“能挡住”）

- 上线 `RestrictedModuleLoader` + `ModuleLoadInterceptor`：先堵住 `require('fs')` / `require('child_process')` 等系统能力。
- `fs` 走替换：尽量让插件用 `plugin.fs`（受限版）替代原生 `fs`。

### 7.2 第二阶段（最小权限）

- 插件开始声明 `ddm.permissions`。
- 默认策略从“宽松兼容”逐步切换为“默认拒绝”（按白名单放行）。

### 7.3 第三阶段（治理完善）

- UI 展示权限（安装/升级时提示）。
- 审计日志与统计（被拒绝次数、访问域名/路径、被拦截模块等）。
- 插件签名/来源校验（官方源/市场）。

# 插件沙箱方案详细设计

## 一、方案概览

针对 Electron 插件系统的沙箱方案，提供三种实现方式，从简单到复杂：

1. **方案一：API 代理沙箱**（推荐，易实施）
2. **方案二：VM2 沙箱**（中等安全，需要额外依赖）
3. **方案三：Worker Threads 隔离**（最安全，但复杂度高）

---

## 二、方案一：API 代理沙箱（推荐）

### 2.1 设计思路

不改变插件执行环境，通过 Proxy 和 API 包装器限制插件可访问的资源。

**优点：**
- 实施简单，无需额外依赖
- 兼容性好，不影响现有插件
- 性能开销小
- 可以渐进式实施

**缺点：**
- 安全性中等（依赖 JavaScript 特性）
- 无法完全隔离（插件仍可访问全局对象）
- **⚠️ 需要额外增强：基础版本无法阻止插件直接 `require('fs')`**

**⚠️ 重要：基础 API 代理沙箱存在安全漏洞！**

插件代码中可以直接 `require('fs')` 来绕过限制。必须使用 `RestrictedModuleLoader` 来拦截 require 调用。

详见：[沙箱方案增强.md](./沙箱方案增强.md)

### 2.2 核心实现

#### 2.2.1 创建受限的文件系统 API

```javascript
// src/main/plugin/sandbox/RestrictedFS.js
import fs from 'fs';
import path from 'path';
import constant from '@/main/common/Constant';

/**
 * 受限的文件系统 API
 * 只允许插件访问自己的目录和指定的共享目录
 */
class RestrictedFS {
    constructor(pluginDir, pluginName) {
        this.pluginDir = pluginDir;
        this.pluginName = pluginName;
        
        // 允许访问的目录白名单
        this.allowedPaths = [
            pluginDir,  // 插件自己的目录
            constant.APPLICATION_DATA_DIR,  // 应用数据目录（只读）
            constant.APPLICATION_TEMP_DIR,   // 临时目录（读写）
        ];
    }

    /**
     * 检查路径是否在允许范围内
     */
    _isPathAllowed(filePath, mode = 'read') {
        const resolvedPath = path.resolve(filePath);
        
        // 检查是否在白名单内
        for (const allowedPath of this.allowedPaths) {
            if (resolvedPath.startsWith(path.resolve(allowedPath))) {
                // 如果是应用数据目录，只允许读
                if (allowedPath === constant.APPLICATION_DATA_DIR && mode === 'write') {
                    return false;
                }
                return true;
            }
        }
        
        // 禁止访问系统目录
        const systemPaths = [
            process.cwd(),
            process.env.HOME,
            '/etc',
            '/usr',
            'C:\\Windows',
            'C:\\Program Files',
        ];
        
        for (const sysPath of systemPaths) {
            if (resolvedPath.startsWith(sysPath)) {
                throw new Error(`插件 ${this.pluginName} 无权访问系统路径: ${resolvedPath}`);
            }
        }
        
        return false;
    }

    /**
     * 读取文件（受限）
     */
    readFileSync(filePath, encoding = 'utf8') {
        if (!this._isPathAllowed(filePath, 'read')) {
            throw new Error(`插件 ${this.pluginName} 无权读取文件: ${filePath}`);
        }
        return fs.readFileSync(filePath, encoding);
    }

    /**
     * 写入文件（受限）
     */
    writeFileSync(filePath, data, encoding = 'utf8') {
        if (!this._isPathAllowed(filePath, 'write')) {
            throw new Error(`插件 ${this.pluginName} 无权写入文件: ${filePath}`);
        }
        return fs.writeFileSync(filePath, data, encoding);
    }

    /**
     * 检查文件是否存在（受限）
     */
    existsSync(filePath) {
        if (!this._isPathAllowed(filePath, 'read')) {
            return false;  // 不抛出错误，只返回 false
        }
        return fs.existsSync(filePath);
    }

    /**
     * 读取目录（受限）
     */
    readdirSync(dirPath) {
        if (!this._isPathAllowed(dirPath, 'read')) {
            throw new Error(`插件 ${this.pluginName} 无权读取目录: ${dirPath}`);
        }
        return fs.readdirSync(dirPath);
    }

    /**
     * 获取文件信息（受限）
     */
    statSync(filePath) {
        if (!this._isPathAllowed(filePath, 'read')) {
            throw new Error(`插件 ${this.pluginName} 无权访问文件: ${filePath}`);
        }
        return fs.statSync(filePath);
    }

    // 其他需要的方法...
}
```

#### 2.2.2 创建受限的网络 API

```javascript
// src/main/plugin/sandbox/RestrictedAxios.js
import axios from 'axios';
import logger from '@/main/common/logger';

/**
 * 受限的网络请求 API
 * 支持域名白名单、请求频率限制
 */
class RestrictedAxios {
    constructor(pluginName, config = {}) {
        this.pluginName = pluginName;
        
        // 默认配置
        this.config = {
            // 允许的域名白名单（空数组表示允许所有）
            allowedDomains: config.allowedDomains || [],
            // 请求超时（毫秒）
            timeout: config.timeout || 10000,
            // 最大请求频率（请求/秒）
            maxRequestsPerSecond: config.maxRequestsPerSecond || 10,
        };

        // 请求频率控制
        this.requestTimestamps = [];
        
        // 创建 axios 实例
        this.instance = axios.create({
            timeout: this.config.timeout,
        });

        // 添加请求拦截器
        this.instance.interceptors.request.use(
            (config) => this._validateRequest(config),
            (error) => Promise.reject(error)
        );

        // 添加响应拦截器
        this.instance.interceptors.response.use(
            (response) => response,
            (error) => {
                logger.error(`插件 ${pluginName} 网络请求失败:`, error.message);
                return Promise.reject(error);
            }
        );
    }

    /**
     * 验证请求是否允许
     */
    _validateRequest(config) {
        const url = config.url || '';
        
        // 检查域名白名单
        if (this.config.allowedDomains.length > 0) {
            try {
                const urlObj = new URL(url);
                const isAllowed = this.config.allowedDomains.some(domain => 
                    urlObj.hostname === domain || urlObj.hostname.endsWith('.' + domain)
                );
                
                if (!isAllowed) {
                    throw new Error(`插件 ${this.pluginName} 无权访问域名: ${urlObj.hostname}`);
                }
            } catch (e) {
                if (e.message.includes('无权访问')) {
                    throw e;
                }
                // URL 解析失败，可能是相对路径，允许
            }
        }

        // 检查请求频率
        const now = Date.now();
        this.requestTimestamps = this.requestTimestamps.filter(
            timestamp => now - timestamp < 1000
        );
        
        if (this.requestTimestamps.length >= this.config.maxRequestsPerSecond) {
            throw new Error(`插件 ${this.pluginName} 请求频率过高，请稍后重试`);
        }
        
        this.requestTimestamps.push(now);
        
        return config;
    }

    // 代理 axios 的方法
    get(url, config) {
        return this.instance.get(url, config);
    }

    post(url, data, config) {
        return this.instance.post(url, data, config);
    }

    put(url, data, config) {
        return this.instance.put(url, data, config);
    }

    delete(url, config) {
        return this.instance.delete(url, config);
    }

    // 其他方法...
}
```

#### 2.2.3 创建沙箱环境

```javascript
// src/main/plugin/sandbox/PluginSandbox.js
import RestrictedFS from './RestrictedFS.js';
import RestrictedAxios from './RestrictedAxios.js';
import logger from '@/main/common/logger';

/**
 * 插件沙箱环境
 * 为插件提供受限的 API 访问
 */
class PluginSandbox {
    constructor(appPlugin, pluginConfig = {}) {
        this.appPlugin = appPlugin;
        this.pluginName = appPlugin.pluginName;
        this.pluginDir = appPlugin.pluginDir;
        
        // 从插件配置或 package.json 读取权限配置
        this.permissions = this._loadPermissions(pluginConfig);
        
        // 创建受限的 API
        this._createRestrictedAPIs();
    }

    /**
     * 加载插件权限配置
     */
    _loadPermissions(pluginConfig) {
        // 从 package.json 的 ddm.permissions 字段读取
        // 或从应用配置读取
        const defaultPermissions = {
            filesystem: {
                read: [this.pluginDir],
                write: [this.pluginDir],
            },
            network: {
                allowedDomains: [],  // 空数组表示允许所有
                maxRequestsPerSecond: 10,
            },
            apis: [],  // 允许调用的 API 列表
        };

        return {
            ...defaultPermissions,
            ...pluginConfig.permissions,
        };
    }

    /**
     * 创建受限的 API
     */
    _createRestrictedAPIs() {
        // 受限的文件系统
        this.fs = new RestrictedFS(this.pluginDir, this.pluginName);
        
        // 受限的网络请求
        this.axios = new RestrictedAxios(this.pluginName, {
            allowedDomains: this.permissions.network.allowedDomains,
            maxRequestsPerSecond: this.permissions.network.maxRequestsPerSecond,
        });

        // 其他受限 API...
    }

    /**
     * 获取沙箱化的插件上下文
     * 替换 AppPlugin 中的危险 API
     */
    getSandboxedContext() {
        // 创建 AppPlugin 的副本，替换危险 API
        const sandboxedPlugin = {
            ...this.appPlugin,
            // 替换文件系统 API
            fs: this.fs,
            // 替换网络 API
            axios: this.axios,
            // 其他受限 API...
        };

        // 冻结对象（防止插件修改）
        Object.freeze(sandboxedPlugin);
        
        return sandboxedPlugin;
    }

    /**
     * 验证 API 调用权限
     */
    checkAPIPermission(moduleName, method) {
        if (this.permissions.apis.length === 0) {
            // 空数组表示允许所有 API
            return true;
        }

        const fullPath = `${moduleName}.${method}`;
        const isAllowed = this.permissions.apis.some(api => {
            if (typeof api === 'string') {
                return fullPath === api || fullPath.startsWith(api + '.');
            }
            // 支持正则表达式
            if (api instanceof RegExp) {
                return api.test(fullPath);
            }
            return false;
        });

        if (!isAllowed) {
            throw new Error(`插件 ${this.pluginName} 无权调用 API: ${moduleName}.${method}`);
        }

        return true;
    }
}
```

#### 2.2.4 修改 PluginLoader 使用沙箱

```javascript
// src/main/plugin/PluginLoader.js (修改版)
import AppPlugin from '@/main/plugin/AppPlugin.js';
import PluginSandbox from '@/main/plugin/sandbox/PluginSandbox.js';
import logger from '@/main/common/logger';
import appConfig from '@/main/common/config/appConfig.js';

class PluginLoader {
    constructor(pluginManager) {
        this.pluginManager = pluginManager;
    }

    async load(pluginName, source = null) {
        try {
            const appPlugin = new AppPlugin(pluginName, source);
            if(!appPlugin.mainPath && !appPlugin.rendererPath && !appPlugin.i18nPath){
                return null;
            }
            
            logger.info(`插件加载[${pluginName}]-开始 pluginPath:${appPlugin.pluginDir}`);
            
            // 读取插件配置（权限设置）
            const pluginConfig = appConfig.getProperty(`pluginConfig.${pluginName}`) || {};
            
            // 创建沙箱环境
            const sandbox = new PluginSandbox(appPlugin, pluginConfig);
            
            // 获取沙箱化的上下文
            const sandboxedPlugin = sandbox.getSandboxedContext();
            
            // 在沙箱环境中加载插件
            if(appPlugin.i18nPath){
                const i18nModule = this.requirePlugin(appPlugin.i18nPath)(sandboxedPlugin.geti18nRegistrar())
                await i18nModule.install();
                logger.info(`插件加载[${pluginName}]-调用i18n install 完毕`);
            }
            
            if(appPlugin.mainPath){
                // 使用沙箱化的插件对象
                const plugin = this.requirePlugin(appPlugin.mainPath)(sandboxedPlugin)
                await plugin.install() 
                logger.info(`插件加载[${pluginName}]-调用main install 完毕`);
            }
            
            // 保存沙箱引用（用于后续权限检查）
            appPlugin._sandbox = sandbox;
            
            return appPlugin;
        } catch (error) {
            logger.error(`插件加载-[${pluginName}]-失败`, error);
            throw new Error(`插件${pluginName}加载失败，错误原因：${error}`)
        }
    }
    
    requirePlugin(mainJs) { 
        delete requireFunc.cache[requireFunc.resolve(mainJs)];
```

## 三、方案二：VM2 沙箱

### 3.1 设计思路

使用 VM2 库在独立的 JavaScript 上下文中执行插件代码。

**优点：**
- 安全性较高
- 可以完全隔离全局对象
- 支持超时控制
- **✅ 可以限制 require，阻止插件直接 require('fs')**

**缺点：**
- 需要额外依赖（vm2）
- **性能开销较大（10-20%）**
- 某些 Node.js API 可能无法使用
- 调试困难
- **⚠️ VM2 已废弃，不再维护**

**VM2 如何限制 require：**

```javascript
const { VM } = require('vm2');
const vm = new VM({
    require: {
        external: false,        // 禁止 require 外部模块
        builtin: ['path', 'url'], // 只允许这些内置模块
        mock: {
            fs: restrictedFS,   // 用受限版本替换
        }
    }
});
```

**VM2 性能开销 10-20% 的原因：**

1. **代码解析和编译**（5-8%）
   - 需要将代码字符串解析成 AST
   - 编译成可执行代码
   - 每次执行都需要这个过程

2. **Proxy 拦截开销**（3-5%）
   - 所有对象访问都通过 Proxy
   - 每个属性访问、方法调用都需要拦截
   - 增加了函数调用栈深度

3. **上下文隔离**（2-4%）
   - 创建独立的 JavaScript 上下文
   - 需要复制/代理全局对象
   - 方法调用需要跨上下文传递

4. **安全检查**（1-3%）
   - 每次 require 都需要检查白名单
   - 对象访问需要验证权限
   - 防止原型链污染等安全检查

### 3.2 核心实现

```javascript
// src/main/plugin/sandbox/VM2Sandbox.js
import { VM } from 'vm2';
import logger from '@/main/common/logger';

class VM2Sandbox {
    constructor(appPlugin, pluginConfig = {}) {
        this.appPlugin = appPlugin;
        this.pluginName = appPlugin.pluginName;
        
        // 创建 VM 实例
        this.vm = new VM({
            timeout: pluginConfig.timeout || 5000,  // 5秒超时
            sandbox: this._createSandbox(),
            // 允许的 require 模块
            require: {
                external: false,  // 不允许 require 外部模块
                builtin: ['path', 'url', 'util'],  // 只允许这些内置模块
                root: appPlugin.pluginDir,  // 插件目录作为根
            },
        });
    }

    /**
     * 创建沙箱环境
     */
    _createSandbox() {
        return {
            // 插件对象（受限版本）
            plugin: this._createRestrictedPlugin(),
            
            // 受限的 console
            console: {
                log: (...args) => logger.info(`[${this.pluginName}]`, ...args),
                error: (...args) => logger.error(`[${this.pluginName}]`, ...args),
                warn: (...args) => logger.warn(`[${this.pluginName}]`, ...args),
            },
            
            // 其他受限 API...
        };
    }

    /**
     * 创建受限的插件对象
     */
    _createRestrictedPlugin() {
        // 只暴露安全的 API
        return {
            name: this.appPlugin.name,
            version: this.appPlugin.version,
            logger: this.appPlugin.logger,
            notify: this.appPlugin.notify,
            // 不暴露 fs, axios 等危险 API
        };
    }

    /**
     * 执行插件代码
     */
    run(code) {
        try {
            return this.vm.run(code);
        } catch (error) {
            logger.error(`插件 ${this.pluginName} 执行失败:`, error);
            throw error;
        }
    }
}
```

**使用方式：**

```javascript

const pluginModule = sandbox.run(code);
await pluginModule.install(sandbox._createSandbox().plugin);
```

## 四、方案三：Worker Threads 隔离

### 4.1 设计思路

使用 Node.js Worker Threads 在独立线程中执行插件。

**优点：**
- 安全性最高（完全隔离）
- 真正的进程隔离
- 可以限制资源使用

**缺点：**
- 实现复杂
- 通信开销大
- 需要序列化数据
- 某些 API 无法在 Worker 中使用

### 4.2 核心实现

```javascript
// src/main/plugin/sandbox/WorkerSandbox.js
import { Worker } from 'worker_threads';
import path from 'path';
import logger from '@/main/common/logger';

class WorkerSandbox {
    constructor(appPlugin, pluginConfig = {}) {
        this.appPlugin = appPlugin;
        this.pluginName = appPlugin.pluginName;
        this.worker = null;
    }

    /**
     * 启动 Worker
     */
    async start() {
        const workerPath = path.join(__dirname, 'plugin-worker.js');
        
        this.worker = new Worker(workerPath, {
            workerData: {
                pluginDir: this.appPlugin.pluginDir,
                pluginName: this.appPlugin.pluginName,
                mainPath: this.appPlugin.mainPath,
            },
        });

        // 监听消息
        this.worker.on('message', (msg) => {
            this._handleMessage(msg);
        });

        this.worker.on('error', (error) => {
            logger.error(`插件 ${this.pluginName} Worker 错误:`, error);
        });

        this.worker.on('exit', (code) => {
            logger.info(`插件 ${this.pluginName} Worker 退出，代码: ${code}`);
        });
    }

    /**
     * 处理 Worker 消息
     */
    _handleMessage(msg) {
        switch (msg.type) {
            case 'api-call':
                // 处理插件发起的 API 调用
                this._handleAPICall(msg);
                break;
            case 'log':
                logger.info(`[${this.pluginName}]`, msg.data);
                break;
        }
    }

    /**
     * 调用插件方法
     */
    async call(method, args) {
        return new Promise((resolve, reject) => {
            const id = Date.now();
            
            const handler = (msg) => {
                if (msg.id === id) {
                    this.worker.removeListener('message', handler);
                    if (msg.error) {
                        reject(new Error(msg.error));
                    } else {
                        resolve(msg.result);
                    }
                }
            };
            
            this.worker.on('message', handler);
            this.worker.postMessage({ type: 'call', method, args, id });
        });
    }

    /**
     * 停止 Worker
     */
    async stop() {
        if (this.worker) {
            await this.worker.terminate();
        }
    }
}
```
